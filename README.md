# 一、代码功能完整总结（mel.py）
这份脚本是墨尔本电动汽车充电站负荷数据集预处理与探索性分析工具，核心作用：清洗、整合多源时序数据，筛选优质站点，构造单站建模特征面板，为后续联邦负荷预测提供标准化输入数据。
## 核心功能拆解
1. 参数入口
支持命令行指定数据路径、目标站点、选取Top-K站点、随机种子；可分析全部站点概况，也可单独提取某一个充电站数据集。

2. 多源数据加载与时序对齐
读取6类原始文件：
`volume.csv`（充电负荷标签）、`weather.csv`（气象特征）、`e_price/s_price.csv`（电价）、`sites/info/chargers.csv`（站点静态信息）；
统一转为**Datetime索引**，完成时序排序、缺失值基础插值填充。

3. 站点质量评级 `rank_stations()`
对每个充电站计算统计指标：均值、标准差、最小值、最大值、零负荷占比、缺失率；
按平均负荷降序排序，筛选出高流量优质站点；同时识别低质量站点（波动极小 / 零负荷占比过高），方便后续实验剔除。

4. 单站点特征面板构建 `build_station_frame()`
选定某一个充电站，拼接：
- 预测目标：`target_volume` 充电负荷
- 外部特征：电价、气象数据
- 手工时序特征：小时正弦余弦、星期、月份周期性编码、周末标记
输出一份整齐的单站时序特征表（panel.csv），这就是联邦训练单客户端的基础数据集。

5. 探索性数据分析EDA
- 输出数据集整体概况：时间跨度、样本数量、站点总数；
- 打印Top高负荷站点清单、低质量站点清单；
- 计算特征与负荷目标的相关系数，初步筛选强相关特征；

6. 结果持久化
自动在`outputs`文件夹输出三类文件：
- `melbourne_station_ranking.csv`：全部站点负荷统计排名
- `melbourne_station_{station_id}_panel.csv`：单站完整特征时序表
- `melbourne_summary.txt`：数据集基本信息日志
